Algorithm

1. Leetcode 167. 两数之和 II - 输入有序数组

题目: 给定一个已按照升序排列 的有序数组，找到两个数使得它们相加之和等于目标数。函数应该返回这两个下标值 index1 和 index2，其中 index1 必须小于 index2。

说明: 返回的下标值（index1 和 index2）不是从零开始的。你可以假设每个输入只对应唯一的答案，而且你不可以重复使用相同的元素。
示例: 输入: numbers = [2, 7, 11, 15], target = 9
输出: [1,2]
解释: 2 与 7 之和等于目标数 9 。因此 index1 = 1, index2 = 2 。

代码：
public int[] twoSumIndex(int[] nums, int target) {
    int left = 0, right = nums.length - 1;
    while (left < right) {
        int sum = nums[left] + nums[right];
        if (sum == target) {
            // 题目要求的索引是从 1 开始的
            return new int[]{left + 1, right + 1};
        } else if (sum < target) {
            left++; // 让 sum 大一点
        } else if (sum > target) {
            right--; // 让 sum 小一点
        }
    }
    return new int[]{-1, -1};
}

总结
只要是数组有序，就可以使用双指针解法！

Review

Tip

最近在与后台交互时的启发
在接收到后台数据之后需要进行排序与整合操作，然后进行显示。
首先后台返回的数据，我们一定是不信任的，需要判断是否为空，或者是否正常。
然后就是这次收获，在对数据进行加工存储的过程中，不按照后台返回的格式，而是自己进行一次封装，
将需要的数据格式自己控制，这样不论后台的数据怎么变化，只要是我这边需要显示和操作的逻辑不变就不需要改变数据，同时，我是访问数据之后提供给其他人使用的，这样，后台修改了返回给我的数据结构，只要是我这边进行整理，其他使用我二次封装数据的地方都不需要进行修改，这样数据的灵活性有一定保障，后台修改不会引起大量的移动端修改。
